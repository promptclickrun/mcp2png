Why it exists

Apps are visual. Clipboard payloads should be too.

MCP Apps can render beautiful UI inside a chat host. When users copy them into Teams, email, docs, or tickets, they need the rendered result. They do not need sandbox HTML, iframe data, or a broken placeholder.

Card copy

Capture a rendered app or host card as a PNG and write it directly to the clipboard with a download fallback.

Message copy

Clone a full chat message and replace MCP App regions with inline PNGs while preserving surrounding prose, tables, and code.

Visual fidelity

Message-copy defaults to visible DOM capture, so the pasted image matches the app card that was on screen.

API

Small primitives, clean host integration.

Use low-level capture helpers, one-call card copy, or rich full-message copy with inline replacement.

Capture

elementToPngBlob and elementToPngDataUrl produce PNG output from visible DOM.

Copy

copyElementAsPng writes an image clipboard item or downloads the file as fallback.

Replace inline

copyMessageElementWithInlineMcpApps preserves message order and swaps app nodes for PNG images.

import { copyMessageElementWithInlineMcpApps } from 'mcp2png';

const message = document.querySelector('[data-chat-message]');

copyButton.addEventListener('click', async () => {
  await copyMessageElementWithInlineMcpApps(
    message,
    '[data-mcp2png-export]',
    {
      backgroundColor: '#ffffff',
      pixelRatio: 2,
    },
  );
});
Fallback model

Built for real paste targets.

Clipboard APIs vary by browser, security context, and destination app. mcp2png keeps the strategy explicit.

1. Full message

Preserve the chat message structure and replace MCP App nodes in place with visible PNG captures.

2. Trusted export

Use app-provided export hooks for card-level copy, or for message copy only when they guarantee visual equivalence.

3. Download fallback

When clipboard writes fail, download a PNG instead of dumping raw app data into the pasteboard.

Proof

The pasted image matches the visible card.

The proof demo extracts the inline PNG from the rich clipboard payload and compares it with the visible MCP App card.

The core is framework-agnostic. React helpers are available from mcp2png/react.
